# Лабораторная 3 — базовые растровые алгоритмы

Требуется показать работу четырёх базовых алгоритмов: пошаговый (step), алгоритм ЦДА (DDA), алгоритм Брезенхэма для отрезков и для окружности. Нужен интерфейс с координатами/осью/сеткой и краткий отчёт с замерами времени.

## Что сделаем
- GUI на Tkinter с холстом и сеткой. Отрисовка пикселей на дискретной сетке, оси и подписи.
- Алгоритмы: step, DDA, Bresenham (отрезок), Bresenham (окружность).
- Управление: ввод координат начала/конца, радиуса, выбор алгоритма, выбор цвета, масштаб клетки.
- Замер времени выполнения выбранного алгоритма и количество пикселей.
- Возможность очистить холст или нарисовать все четыре алгоритма для сравнения (опционально по кнопке).

## Как запускать
```bash
python app.py
```

Python 3 + Tkinter (обычно в пакете `python3-tk`).

## Как пользоваться
- Выберите алгоритм (step, DDA, Bresenham отрезок, Bresenham окружность).
- Задайте координаты X0,Y0,X1,Y1 (для окружности используются X0,Y0 как центр и R как радиус).
- Подстройте размер клетки и сетки, чтобы точки были видны целиком.
- Нажмите «Нарисовать», ниже появится число пикселей и время работы алгоритма.
- «Очистить» — сброс холста.

## Структура
- `app.py` — реализация алгоритмов и GUI.
- `README.md` — описание.

## Пояснения по алгоритмам
- **Step**: идём по максимальному из |dx|, |dy| с равномерным шагом, координаты округляем до целых.
- **DDA (ЦДА)**: накапливаем дробные x/y с постоянным шагом, каждую итерацию берём округлённые координаты.
- **Брезенхэм (отрезок)**: целочисленный алгоритм с ошибкой, минимизирует число операций и выбирает пиксель из двух соседей.
- **Брезенхэм (окружность)**: рисуем дугу в первой восьмой и отражаем в остальные октанты.
