# Лабораторная 2 — обработка изображений (вариант 11)

Вариант 11 из таблицы — **локальная пороговая обработка** (нужно реализовать два метода на выбор и сравнить их работу).

## Что требуется по заданию
- Графический интерфейс.
- База соответствующих тестовых изображений (размытые, шумные, мало-контрастные и т.п.) под выбранные методы.
- Два метода локальной пороговой обработки, сравнить результаты и пояснить, где какой метод лучше.
- Пояснения по работе методов и подбору изображений.

## План по реализации
- Реализовать два локальных пороговых алгоритма, например:  
  - Adaptive Mean / Adaptive Gaussian (OpenCV аналоги)  
  - Sauvola / Niblack (более устойчивы к неравномерному освещению)  
  Выбери любые два, но опиши, почему именно они.
- Показать исходное изображение, результат для каждого метода и/или разницу.
- Дать пользователю управление ключевыми параметрами (радиус окна, k для Sauvola/Niblack, порог смещения и т.д.).
- Сделать режим сравнения на нескольких тестовых картинках.

## Какие изображения нужны
Цель — показать преимущество локальных пороговых методов на проблемных кейсах. Подойдут:
- Фото документа/книги при косом или неравномерном освещении (тени, засветы).
- Скан/фото с низким контрастом или лёгким блюром (смартфон).
- Текстура с плавным градиентом яркости (стена, бумага, ткань).
- Любая сцена с тенями, где глобальный порог даёт «злипшиеся» области.

Где взять:
- Самые быстрые: сделать 3–5 собственных фото/сканов на телефон — этого достаточно для отчёта.
- Готовые датасеты: DIBCO/DocImg (документные снимки с тенями), типичные образцы есть на Kaggle или в свободных подборках по «document binarization dataset» / «shadowed text image».
- Можно взять публичные примеры из статей/репозиториев про adaptive thresholding и сохранить их локально.

Положи файлы в `Lab_2/images/`, назови осмысленно (например, `doc_shadow.jpg`, `low_contrast.png`, `blur_text.jpg`, `gradient_texture.jpg`).

## Структура
- `app.py` — GUI: выбор картинки, Adaptive Gaussian и Sauvola, настройки окна/порогов.
- `generate_report.py` — скрипт, который прогоняет тесты и собирает PDF.
- `images/` — тестовые входные изображения.
- `results/` — сохранённые выходы и PDF-отчёт.
